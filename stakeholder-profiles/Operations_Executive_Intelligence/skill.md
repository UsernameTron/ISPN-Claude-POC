## How to Request Analysis

### Basic Initiative Portfolio Review

**Request:**
> "Generate initiative portfolio dashboard from current Asana projects. Include health status, resource utilization, and executive escalation items."

**What Claude Does:**
1. Connects to Asana MCP to pull all strategic initiative project data
2. Analyzes task completion rates, milestone adherence, and resource allocation
3. Identifies projects requiring executive attention (behind schedule, over budget, blocked)
4. Creates executive dashboard with color-coded status and action recommendations
5. Generates executive summary with 3-5 key decisions required

**Time Required:** 3-5 minutes for complete portfolio analysis

### Deep-Dive Resource Allocation Analysis

**Request:**
> "Analyze resource allocation across all strategic initiatives. Identify over-allocation, conflicts, and rebalancing opportunities. Create recommendations for Q1 resource optimization."

**What Claude Does:**
1. Maps all initiative resource requirements from Asana project data
2. Calculates current and projected resource utilization by department
3. Identifies resource conflicts and over-allocation scenarios
4. Models resource rebalancing scenarios with timeline impacts

## Best Practices

1. **Daily Operational Pulse**: Run basic portfolio health check every Monday morning to identify emerging issues early.

2. **Weekly Executive Prep**: Generate executive team sync materials every Friday with upcoming week priorities and decision items.

3. **Monthly Board Preparation**: Create board operational review materials first Monday of each month, allowing time for executive review and refinement.

4. **Quarterly Portfolio Review**: Conduct comprehensive initiative portfolio optimization quarterly, including resource rebalancing and strategic prioritization.

5. **Exception-Based Management**: Configure alerts for initiatives falling below performance thresholds (timeline, budget, resource utilization).

## Common Questions

**Q: How does Claude identify which initiatives require executive escalation?**  
A: Claude applies escalation criteria based on: (1) Timeline delays >2 weeks, (2) Budget overruns >10%, (3) Resource conflicts affecting multiple initiatives, (4) Cross-functional bottlenecks requiring executive authority, (5) Strategic value at risk >$100K.

**Q: Can Claude analyze initiatives across different Asana workspaces?**  
A: Yes, Claude can aggregate data from multiple Asana workspaces if MCP access is configured. Best practice is to maintain consistent project naming and status conventions across workspaces.

**Q: What if operational data exists outside Asana?**  
A: Claude can incorporate data from files stored in Filesystem MCP locations. Upload CSV/XLSX files to designated folders (e.g., `/Operations/External-Data/`) and Claude will integrate during analysis.

**Q: How does Claude handle confidential or sensitive operational data?**  
A: All processing occurs locally through MCP connections. No data is transmitted externally. Executive dashboards can be generated with different sensitivity levels (board-appropriate vs. internal team details).
